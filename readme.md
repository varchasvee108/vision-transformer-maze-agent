# Maze World Model

A Vision Transformer based navigation system designed to solve procedurally generated mazes by predicting optimal actions from visual inputs.

## Overview

The Maze World Model project implements an intelligent navigation agent that processes raw imagery of a maze to determine the best path to an exit. Unlike standard reactive agents, this project focuses on building an **algorithmic vision model**. 

The core objective is to determine if a Vision Transformer can learn to replicate the logic of a **(BFS)** algorithm through visual perception alone. By training on optimal paths generated by BFS, the model attempts to "solve" the maze topology in a single forward pass, essentially internalizing the shortest-path algorithm within its attention heads.

## Core Features

*   **Dual-Algorithm Data Generation**:
    *   **DFS Generation**: Uses a recursive DFS algorithm to carve out complex, perfect mazes (no loops).
    *   **BFS Ground-Truth**: Employs a BFS solver to calculate the mathematically optimal action from every single cell to the exit, providing a rich, dense training signal.
*   **Vision Transformer Architecture**: Employs a custom Patch-based Transformer that treats maze segments as visual tokens.
*   **Attention Rollout Visualization**: Offers transparency into the model's decision-making by generating heatmaps that show where the agent is focusing its attention.
*   **Lightning-Optimized Training**: Built on PyTorch Lightning for efficient, scalable training with multi-GPU and mixed-precision support.
*   **Trajectory Evaluation**: Automated rollout scripts with oscillation detection to prevent infinite loops and validate navigation success.

## Technical Architecture

The system follows a Vision Transformer pipeline designed to emulate algorithmic reasoning:
1.  **Maze Synthesis (DFS)**: A randomized Depth-First Search creates the spatial environment.
2.  **Policy Labeling (BFS)**: A BFS pass generates the ground-truth labels, ensuring the model learns the minimal distance path.
3.  **Visual Patching**: Converts $132 \times 132$ maze images into discrete $11 \times 11$ patches.
4.  **Position Encoding**: Injects absolute spatial context so the model can relate distant walls and exits.
5.  **Self-Attention Blocks**: Multiple layers of Multi-Head Self-Attention process the visual sequence, effectively "tracing" paths across tokens.
6.  **Action Prediction**: Maps the final CLS token to four primary directions: Up, Down, Left, or Right.

## Getting Started

### Installation

Clone the repository and set up the environment:

```bash
make setup
```

### Data Generation

Generate a new dataset of procedural mazes:

```bash
make generate-data
```

### Training

Start the training process:

```bash
make train
```

### Visualization

To run agent rollouts and see the attention heatmaps in action:

```bash
python -m scripts.visual_attn
```

## Project Layout

*   `core/`: Configuration management and core system logic.
*   `data/`: Data generation, processing, and visual rendering modules.
*   `models/`: ViT architecture and custom neural layers.
*   `maze_dataset/`: PyTorch dataset implementations for vision tasks.
*   `scripts/`: Central entry points for training and visualization.
*   `visuals/`: Repository for generated rollout frames and heatmaps.
